## unsigned 자료형
unsigned 변수에 큰 양수를 저장하고 %d로 출력하면 음수가 출력될 가능성이 있으며 음수를 저장하고 %u로 출력하면 양수가 출력된다.  
[unsigned를 잘못 사용한 경우](./3-5.c)
실행결과  
-1  
4294967295  

4294967295는 2진수로 : `11111111 11111111 11111111 11111111`  
음수는 그 절대값을 2의 보수로 저장하므로  
-1도 2진수로 : `11111111 11111111 11111111 11111111`로 메모리에 저장된다.  
printf함수에 %d를 사용하여 출력하면 메모리에 저장된 비트열의 가장 왼쪽 비트를 부호비트로 간주하므로 -1을 출력하지만 %u를 사용하면 부호비트를 고려하지 않고 모든 비트를 10진수로 바꾸므로 4294967295를 출력한다.

## 실수 자료형
- float : 크기4, 유효숫자7, 변환문자 %f
- **double** : 크기8, 유효숫자15, 변환문자 %lf
- long double : 크기8이상(컴파일러마다 다름), 유효숫자 15이상, 변환문자 %Lf

정수형을 기본으로 사용하고 실수형은 꼭 필요한 경우만 사용한다.
실수형은 유효숫자가 많은 **double형을 기본**으로 사용한다.

## 문자열 저장
문자열은 char형을 배열 형태로 만들어 저장한다.
```c
char fruit[6] = "apple";
```
컴파일러가 문자열의 끝을 표시하기 위해 null문자('\0', 아스키코드0)를 자동으로 추가하기 때문에 배열의 크기는 문자열의 길이보다 1이상 크게 잡아야 한다.  
[문자열의 길이보다 1이상 크게 안잡은 경우](./exercise1.c)  
warning : initializer-string for array of chars is too long  
결과 : banan

``` c
char fruit[20] = "strawberry";
fruit = "banana" // 배열은 선언 된 이후에 대입연산자로 문자열을 입력할 수 없다.
```

**char 배열에 초기화된 문자열을 다른 문자열로 바꾸려면 `strcpy`함수를 사용한다.**
